# Mr. Problem Focus

## Core Strength

Mr. Problem is strongest when a user has a fuzzy or overconfident idea and needs a disciplined path from conversation to durable artifacts.

The core capability is not "use every discovery framework." The core capability is:

1. preserve problem evidence,
2. expose hidden dimensions,
3. keep human judgment visible,
4. route to the right artifact,
5. validate each handoff.

## Double Down

Double down on the parts that make Mr. Problem different:

- **Artifact routing.** Decide whether the next durable thing is a problem map, stakeholder map, requirements matrix, ADR, C4 view, or PRD.
- **Traceability.** Every output should point back to evidence, a collapsed dimension, an issue, a stakeholder position, an ADR, or a validation example.
- **Residue protection.** The system should name what remains human judgment instead of pretending it has been automated away.
- **Convergence discipline.** Each step should end in a small, inspectable artifact with a clear owner and next move.
- **Validation examples.** Every artifact shape needs examples that define what good output looks like and what failure looks like.

## Lose The Extracurriculars

Do not expand the repository by adding every adjacent methodology as a top-level skill.

Avoid:

- generic design thinking methods without a durable artifact,
- full design sprint facilitation,
- broad product management coaching,
- research frameworks that do not change a handoff,
- diagramming for its own sake,
- benchmarking rituals that do not catch real regressions.

## Adoption Rule

Adopt a methodology only when it strengthens one of Mr. Problem's core artifacts.

| Method | Adopt As | Do Not Adopt As |
| --- | --- | --- |
| Opportunity Solution Trees | Requirements matrix shaping | A separate product discovery universe |
| IBIS dialog mapping | Stakeholder issue/position map | General meeting notes |
| C4 model | Architecture view attached to ADR/PRD | Diagram collection |
| SkillWiki | Provenance ledger for skills and artifacts | Academic ontology project |
| SWE-Skills-Bench | Validation example discipline | Leaderboard optimization |
